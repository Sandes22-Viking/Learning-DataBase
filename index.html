What is a database?

Database is a organized collection of structured information, typically stored electronically in a computer system. A database is usually controlled by DBMS. Along with the applications that are associated with them, they're called of database system

Why use a database?

  • Databases can store very large numbers of records efficiently (they take up little space).
  • It is very quick and easy to find information.
  • It is easy to add new data and to edit or delete old data.
  • Data can be searched easily, eg 'find all Ford cars'.
  • Data can be sorted easily, for example into 'date first registered' order.
  • Data can be imported into other applications, for example a mail-merge letter to a customer saying that an MOT test is due.
  • More than one person can access the same database at the same time - multi-access.
  • Security may be better than in paper files.

Database uses

When a database holds details about people, it's likely to include their first name, surname and their date of birth. In addition to this, specialist information is stored depending on the database's intended use.

Organisations use databases for a variety of reasons:

  • the police have details of all known criminals in a database, eg crimes they've comitted.
  • schools use a database to store details about their pupils, eg how many days they've been off school sick.
  • a hospital will store details of all its patients in a database, eg a history of their health issues.
  • the Government uses a database to store records of people's income tax payments.
  • a database is used to keep track of all the drivers in central London who have (or haven't) paid the Congestion Charge.

First Steps with MySQL

At first, you must create an element. With the language is very easy to understand. For exemple:

CREATE DATABASE databaseTest
default character set utf8
default collate utf8_general_ci;

And then you must create a call to use the databaseTest. Use this element:

USE databaseTest;

Creating Tables

CREATE TABLE providers(
codigo int(4) AUTO_INCREMENT,
nome varchar(30) NOT NULL,
email varchar(50),
PRIMARY KEY (codigo) );

Now for the explanations of what the commands are about:

  • AUTO_INCREMENT can be used to automate code that serves as the primary key of a table.
  • PRIMARY KEY defines the primary key of the table, that is, the field that serves as the key of the table and that cannot be repeated.
  • NOT NULL defines that a given field is mandatory.

Now we already have a database and a table created, so it is possible to manipulate the data in the database.

Insert Information in the Table

Now we are going to insert some information in our table, for that we'll use the INSERT command.

  • Primary Key

    • The PRIMARY KEY constraint uniquely identifies each record in a table. Primary keys must contain UNIQUE values, and cannot contain NULL values. A table can have only ONE primary key; and in the table, this primary key can consist of single or multiple columns (fields).

    CREATE TABLE providers(
    codigo int(4) AUTO_INCREMENT,
    nome varchar(30) NOT NULL,
    email varchar(50),
    PRIMARY KEY (codigo) );
  • Describe

    • The DESCRIBE statement is used more to obtain information about a table structure while EXPLAIN statement is used to obtain a query execution plan.

    DESCRIBE table providers;
  • Alter

    • The alter command is used when we want to modify a database or any object contained in the database.

    It is possible to change the structure of a table in MySQL after it has been created, adding or deleting attributes (fields). We use the ALTER TABLE command for this

    ALTER TABLE providers
    DROP COLUMN email;

    With this DROP COLUMN you can delete a column of your table.

    ALTER TABLE providers
    ADD COLUMN email;

    And the same, you can add a column.

    You can exclude a primary key (or other constraint)

    ALTER TABLE providers
    DROP primary key;
  • Update

    • The update command has a syntax very similar to that of INSERT, but it is used to update database records.

    UPDATE providers
    SET email = 'erick-sandes@hotmail.com'
    WHERE codigo = '2';

    With this command there will be an update, modifying the desired part located by the primary key.

  • Delete

    • The delete command should never be used without the WHERE clause. Otherwise, it will delete all the records in the table. Whenever you want to delete a record, you probably want to delete one in particular. Thus, it is always important to use WHERE, as shown in the example below.

    DELETE FROM providers
    WHERE id = 1;

    In this way, you will be deleting the record that contains id 1.

  • Truncate

    • The TRUNCATE TABLE command allows you to remove all rows from a table in a single operation, without recording deletions for individual rows. The TRUNCATE TABLE is equivalent to executing the DELETE statement, but without using the WHERE clause. Therefore, it is used to completely delete the contents of a table in MySQL.

    TRUNCATE TABLE providers;
  • Delete Tables

    • The DROP TABLE statement allows a table to be removed from a MySQL database. This statement deletes the entire structure as well as the content of the table.

    DROP TABLE providers;
  • Insert

    • The INSERT INTO statement is used to insert new records in a table.

    INSERT INTO table_name VALUES (value1, value2, value3, ...);
  • Select

    • The select command is quite simple and, for you, it shows the data in the table. Its syntax can be used in some ways, such as:

    SELECT * FROM TABLE_NAME;
  • Clausule Where

    • It makes it possible to restrict the data that are required, making it possible for them to appear in the search.
    Logical Operators:

    • = equal to
    • > greater than
    • > = greater than or equal to
    • < less than
    • <= less than or equal to different Examples: Show datas where cli_codigo is> = 2:

    select * from providers
    where codigo >= 2;

    with this we will select all data with the primary code greater than or equal to 2.

  • Clausule Order By

    • Changes the order in which the search results are presented, enabling them to be placed in ascending and descending order. Examples: Sort by the cli_name attribute (in ascending alphabetical order):

    select * from providers order by nome;

    with that, we will sort all registered names, in alphabetical order.
    It is also possible to define an ascending or descending order. For this we will use the DESC and ASC commands

    select * from providers order by nome asc;
    or
    select * from providers order by nome desc;

  • Between and / Not Between

    • Lists data between two values. Examples: Display data where the code is between 2 and 3:
